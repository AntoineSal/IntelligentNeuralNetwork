\begin{abstract}
The dominance of Transformer architectures has shifted the focus of deep learning from modular, biologically-inspired designs to monolithic, static layers optimized for parallel computation. While highly effective, this approach sacrifices the dynamic routing and functional specialization inherent to biological intelligence ("System 2" thinking). In this work, we propose the \textbf{Intelligent Neurons Network (INN)}, a novel architecture that reimagines the neural network as a graph of stateful agents interacting via a differentiable attention mechanism. 
We introduce \textbf{INNv2}, a scalable implementation leveraging Selective State Space Models (Mamba) to achieve linear inference complexity $O(L)$ and parallelizable training. We demonstrate that INNv2 converges \textbf{comparably to Transformer baselines} on character-level language modeling tasks (reaching 1.16 validation loss on Tiny Shakespeare), while exhibiting emergent modularity on multi-task arithmetic/literary benchmarks. Our results suggest that decoupling memory (SSM) from routing (Attention) is a viable path toward more interpretable and adaptive architectures.
\end{abstract}

\section{Introduction}

Contemporary Deep Learning is dominated by the Transformer architecture \citep{vaswani2017attention}. While remarkably successful, Transformers operate as static computational graphs: every token is processed by the entire network depth, regardless of the task's complexity. This contrasts with biological neural networks, which exhibit dynamic routing, recruiting specialized sub-circuits on demand \citep{kahneman2011thinking}.

We argue that the next step in architectural evolution lies in bridging the gap between \textbf{Graph Neural Networks (GNNs)} (modular, dynamic) and \textbf{Sequence Models} (scalable, parallel). We introduce the \textbf{Intelligent Neuron}, a computational unit defined by:
\begin{enumerate}
    \item \textbf{Local Memory:} A compressed history of the signal (via State Space Models).
    \item \textbf{Selective Communication:} A learnable interface (Keys/Queries) to route information dynamically.
\end{enumerate}

Our contribution, \textbf{INNv2}, replaces the traditional Multi-Layer Perceptron (MLP) layers of Transformers with a population of these independent Mamba-based neurons, interacting via a global "Neuron Attention" mechanism. This yields an architecture that is both biologically plausible and computationally efficient on modern hardware.

\section{Related Work}

\paragraph{Recurrent Neural Networks (RNNs).}
Traditional RNNs like LSTM \citep{hochreiter1997long} capture temporal dynamics but suffer from sequential training bottlenecks ($O(L)$ time complexity). Our INNv1 architecture utilized LSTMs, confirming the functional viability of dynamic routing but revealing severe scalability limits.

\paragraph{Transformers and Attention.}
Transformers solve the training bottleneck via parallelism but incur quadratic inference costs ($O(L^2)$) with respect to sequence length. Several works have attempted to linearize attention (Linear Transformers, Performer) or sparse attention patterns. Our work differs by applying attention \textit{across the neuron dimension} ($N$) rather than the time dimension ($L$).

\paragraph{State Space Models (SSMs).}
Structured State Space Models like S4 \citep{gu2021efficiently} and Mamba \citep{gu2023mamba} offer linear-time scaling ($O(L)$) and parallel training. Mamba introduces input-dependent selection mechanisms. INNv2 adopts Mamba as the internal "engine" of each neuron, effectively parallelizing the memory update.

\paragraph{Dynamic Routing & MoE.}
Mixture of Experts (MoE) \citep{shazeer2017outrageously} introduces sparsity by routing tokens to specific experts. INNv2 can be seen as a fine-grained, differentiable MoE where every neuron is a potential expert, and routing is determined by a continuous attention matrix rather than a discrete router.

\section{Methodology: The INN Architecture}

The INNv2 architecture consists of a stack of $K$ layers. Each layer alternates between two phases: \textbf{Intra-Neuron Processing} (Time-Mixing) and \textbf{Inter-Neuron Communication} (Channel-Mixing).

\subsection{Micro-Architecture: The Mamba Neuron}
Let $N$ be the number of neurons per layer, and $d$ the dimension of each neuron's state.
Each neuron $i \in \{1, \dots, N\}$ processes its temporal input stream $\bx^{(i)} \in \Real^{L \times d}$ independently using a Mamba Block.

The underlying mechanism is a discretized State Space Model:
\begin{align}
    \bh'_t &= \bA \bh'_{t-1} + \bB \bx_t \\
    \mathbf{y}_t &= \bC \bh'_t + \bD \bx_t
\end{align}
In the Mamba implementation, the matrices $\bB, \bC, \mathbf{\Delta}$ (step size) are functions of the input $\bx_t$, allowing the neuron to selectively ignore or store information.
This operation is parallelized across the time dimension $L$ via a parallel scan algorithm \citep{gu2023mamba}.

\subsection{Macro-Architecture: Dynamic Routing (Neuron Attention)}
To allow information flow between neurons, we introduce a global mixing step. Unlike Transformers which mix information across time (Self-Attention), INNv2 mixes information across the neuron population.

At each layer, given the state of all neurons $\mathbf{H} \in \Real^{N \times L \times d}$:
\begin{enumerate}
    \item We permute dimensions to treat neurons as the sequence: $\mathbf{H}_{perm} \in \Real^{L \times N \times d}$.
    \item We apply Multi-Head Attention (MHA) where the "sequence length" is $N$:
    \begin{equation}
        \text{Attention}(Q, K, V) = \softmax\left(\frac{QK^T}{\sqrt{d}}\right)V
    \end{equation}
    Here, $Q, K, V$ are projections of the neuron states.
\end{enumerate}

This mechanism allows a neuron $i$ to "read" the memory of neuron $j$ if their content keys match, effectively creating a dynamic graph topology that evolves at each layer.

\subsection{Algorithm}

\begin{algorithm}[H]
\caption{Forward Pass of INNv2 Layer}
\begin{algorithmic}[1]
\STATE \textbf{Input:} $\mathbf{X} \in \Real^{B \times N \times L \times d}$
\STATE \textit{// Phase 1: Independent Thought (Mamba)}
\STATE $\mathbf{X}_{res} \leftarrow \mathbf{X}$
\STATE $\mathbf{X} \leftarrow \text{VectorizedMambaBlock}(\mathbf{X})$ \COMMENT{Parallel across B, N, L}
\STATE $\mathbf{X} \leftarrow \text{LayerNorm}(\mathbf{X} + \mathbf{X}_{res})$
\STATE \textit{// Phase 2: Communication (Attention)}
\STATE $\mathbf{X}_{flat} \leftarrow \text{Rearrange}(\mathbf{X}, 'b\ n\ l\ d \to (b\ l)\ n\ d')$
\STATE $\mathbf{X}_{comm} \leftarrow \text{MultiHeadAttention}(\mathbf{X}_{flat})$ \COMMENT{Mixes across N}
\STATE $\mathbf{X}_{comm} \leftarrow \text{Rearrange}(\mathbf{X}_{comm}, '(b\ l)\ n\ d \to b\ n\ l\ d')$
\STATE $\mathbf{X} \leftarrow \text{LayerNorm}(\mathbf{X}_{comm} + \mathbf{X})$
\STATE \textbf{Return} $\mathbf{X}$
\end{algorithmic}
\end{algorithm}

\section{Experiments}

\subsection{Experimental Setup}
We implemented INNv2 in PyTorch using a custom vectorized Mamba block. All experiments were run on a single NVIDIA A100 (40GB) GPU.
\begin{itemize}
    \item \textbf{Dataset:} Tiny Shakespeare (1.1M characters).
    \item \textbf{Model Config:} $N=64$ neurons, $d=64$ dimension, 4 layers. Total params $\approx 8.6$M.
    \item \textbf{Optimization:} AdamW, Learning Rate $1e^{-3}$, Batch Size 32, Sequence Length 128.
\end{itemize}

\subsection{Results on Language Modeling}

We compared the convergence speed and final perplexity of INNv2 against our previous LSTM-based implementation (INNv1) and a standard Transformer baseline (nanoGPT configuration, ~10M params).

\begin{table}[h]
    \centering
    \caption{Comparison of Training Efficiency and Performance.}
    \label{tab:results}
    \begin{tabular}{lccccc}
        \toprule
        \textbf{Model} & \textbf{Params} & \textbf{Inference} & \textbf{Time/Epoch} & \textbf{Val Loss} \\
        \midrule

        Transformer & 10.0M & $O(L^2)$ & $\sim$ 250s & 1.48 \\

\textbf{Analysis:} INNv2 outperforms the Transformer baseline on this scale, reaching a validation loss of \textbf{1.16} (Perplexity $\approx 3.19$). This validates that the combination of SSM state compression and dense neuronal communication is highly effective for character-level modeling. Crucially, INNv2 achieves this with a training speed comparable to Transformers, solving the scalability issue of INNv1.

\subsection{Emergent Modularity (Multi-Tasking)}

To test the "System 2" capabilities, we trained a Hybrid INNv1 on a mixed dataset containing both Shakespearean text and arithmetic equations (\texttt{MATH: 378+301=681}).

We observed two key phenomena:
\begin{enumerate}
    \item \textbf{Task Separation:} The attention mechanism activated distinct subsets of "Dynamic Neurons" depending on the input type (Text vs Math).
    \item \textbf{Auxiliary Benefit:} The model trained on the mixed dataset converged to a better loss (1.84) than the model trained on text alone (1.99), suggesting that learning arithmetic structures regularized the latent space.
\end{enumerate}

\section{Discussion and Limitations}

\textbf{Scalability.} While efficient at the 10M parameter scale, scaling laws for INNv2 remain to be tested on billion-token datasets (e.g., The Pile). The $O(N^2)$ complexity of the inter-neuron attention could become a bottleneck if $N$ grows to thousands. Future work will explore sparse or linear attention for the neuron mixing step.

\textbf{Algorithmic Logic.} While the model learned basic arithmetic syntax and approximation, exact long-form calculation remains a challenge for all autoregressive models. The fixed state size of Mamba blocks may limit the ability to "count" over indefinitely long sequences without external memory.

\section{Conclusion}

We presented INNv2, a scalable architecture combining the best of State Space Models and Graph Neural Networks. By treating neurons as independent stateful agents that communicate via attention, we move closer to biologically plausible architectures without sacrificing the hardware efficiency required for modern Deep Learning. The superior convergence on Tiny Shakespeare suggests that "Intelligence Is All You Need"—specifically, the intelligent routing of information between specialized memory units.
